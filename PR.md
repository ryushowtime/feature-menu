# Feature Menu - 功能中心

## 一、项目概述

Feature Menu 是 Claude Code 的插件管理工具，帮助用户浏览和管理已安装的技能（Skills）、代理（Agents）和命令（Commands）。

---

## 二、目标用户

- **主要用户**：所有 Claude Code 用户，包括技术新手
- **用户特点**：
  - 需要快速找到适合当前任务的工具
  - 期望工具直观易用，不需要复杂配置
  - 可能不了解 Claude Code 的全部能力

---

## 三、核心功能

### 3.1 首页 Dashboard
- 显示统计数据：已安装的 Skills、Agents、Commands 数量
- 最近使用的技能列表（带使用次数）
- 收藏夹快捷入口
- Task 推荐入口
- 帮助中心入口

### 3.2 技能浏览（Skills）
- 搜索技能（按名称、描述关键词，仅限当前 Tab）
- 按分类筛选（开发框架、数据库、测试、工作流、业务、DevOps、安全、编程语言、其他），筛选标签可一键移除
- 按名称排序
- 按使用次数排序
- 收藏/取消收藏技能（收藏数据仅针对 Skills）
- 查看技能详情

### 3.3 代理浏览（Agents）
- 查看已安装的代理列表
- 显示每个代理的名称、描述、可用工具、模型
- 代理按名称搜索（不支持分类筛选）
- 无详情弹窗，点击卡片无额外操作

### 3.4 命令浏览（Commands）
- 查看已安装的命令列表
- 显示每个命令的名称、描述
- 点击卡片一键复制命令到剪贴板（无需额外弹窗）

### 3.5 技能详情弹窗
- 显示技能名称、图标、分类、难度等级
- 显示技能描述
- 显示使用场景（When to Use）
- 显示触发命令
- 显示相关技能
- 收藏/取消收藏按钮
- "开始使用"按钮

### 3.6 Task 推荐向导
- 步骤式交互：选择任务类型 → 查看推荐技能
- 任务类型以网格按钮展示（如：前端开发、后端开发、测试、DevOps 等）
- 推荐结果以技能卡片展示
- 点击推荐技能直接打开详情弹窗

### 3.7 帮助弹窗
- 合并术语表（Glossary）和常见问题（FAQ）
- Tab 切换不同内容

### 3.8 新手引导（Onboarding）
- 首次使用时显示 3-4 步图文介绍
- 关键操作首次使用时有高亮指引
- 可选择"不再显示"

### 3.9 空状态设计

#### 无搜索结果
- Skills 页面：显示"没有找到匹配的技能"
- Agents 页面：显示"没有找到匹配的代理"
- Commands 页面：显示"没有找到匹配的命令"

#### 无收藏（仅 Skills）
- 空状态提示："还没有收藏任何技能"
- 引导文案："点击技能卡片的 ☆ 来收藏常用技能"

#### 首次使用（无最近使用、无收藏）
- 显示欢迎引导："从这里开始"
- 提供"浏览技能"和"Task 推荐"两个入口按钮

### 3.10 反馈系统

#### Toast 通知
- 操作成功/失败提示，2-3 秒自动消失
- 收藏成功提示："已添加到收藏夹"
- 取消收藏提示："已从收藏夹移除"
- 复制命令成功提示："已复制到剪贴板"
- 技能使用次数 +1 提示："已记录使用"

#### 确认对话框
- 危险操作（如有）需二次确认

---

## 四、数据管理

### 4.1 数据范围
- **收藏夹**：仅针对 Skills（Agents 和 Commands 不支持收藏）
- **使用记录**：仅针对 Skills（记录每个 Skill 被"使用"的次数）
- **搜索**：按 Tab 隔离（Skills 页面只搜 Skills，Agents 页面只搜 Agents）

### 4.2 存储方式
- 收藏夹数据：浏览器本地存储
- 使用记录：浏览器本地存储
- 无需登录账号
- 每次打开自动扫描本地 Claude Code 目录获取最新数据

---

## 五、技术栈

- **框架**：Next.js 14 (React) - App Router，服务端渲染
- **样式**：Tailwind CSS v3
- **图标**：Lucide React

### 架构约束
- 本地独立应用，非纯前端静态部署
- 服务端直接读取本地文件系统（Skills/Agents/Commands 存储在用户本地）

---

## 六、设计风格

### 6.1 整体美学
**科技感 / Cyberpunk**

- 深色主题为主
- 发光效果、霓虹色调
- 几何线条装饰（可选）
- 现代感字体

### 6.2 色彩方案

| 用途 | 颜色 |
|------|------|
| 主强调色（发光/高亮） | 亮蓝色 #71c4ef |
| 次强调色（图标/标签） | 深青色 #00668c |
| 卡片背景 | 深紫蓝 #1a1a2e |
| 主背景 | 接近纯黑 #0a0a0f |
| 边框 | 深灰色 #3b3c3d |
| 主文字 | 白色 #ffffff |
| 次文字 | 柔和蓝 #b6ccd8 |

### 6.3 辅助色
- 紫色（用于 Agents）：#a855f7 或类似
- 绿色（用于成功状态）：#22c55e
- 黄色（用于警告/收藏）：#eab308
- 红色（用于危险操作）：#ef4444

### 6.4 难度等级配色
- 新手（Newbie）：绿色系
- 进阶（Intermediate）：黄色系
- 高级（Advanced）：红色系

### 6.5 视觉效果
- **发光边框**：卡片悬浮时边框发光
- **渐变装饰**：卡片角落的微妙的渐变装饰
- **阴影扩散**：悬浮时阴影扩散，带发光色调
- **动画过渡**：所有状态变化使用平滑过渡（0.2-0.3s）

## 七、图标要求

### 7.1 图标库
**必须使用**：Lucide React

### 7.2 技能分类图标映射

| 分类 | 图标 | 说明 |
|------|------|------|
| 开发框架 (framework) | Puzzle | 拼图，象征模块组合 |
| 数据库 (database) | Database | 数据库 |
| 测试 (testing) | FlaskConical | 烧瓶，实验室/测试 |
| 工作流 (workflow) | Workflow | 工作流 |
| 业务 (business) | Briefcase | 公文包 |
| DevOps (devops) | Rocket | 火箭，部署/发布 |
| 安全 (security) | Shield | 盾牌 |
| 编程语言 (language) | Code | 代码 |
| 其他 (other) | Box | 通用容器 |

### 7.3 功能图标
- 导航图标：BookOpen（所有技能）、Bot（代理）、Terminal（命令）
- 操作图标：Star（收藏）、Copy（复制）、Sparkles（Task 推荐）、HelpCircle（帮助）

### 7.4 图标风格
- 线条风格，2px 描边
- 尺寸：16px（small）、20px（medium）、32px（large）
- 颜色：跟随上下文（分类图标用 text-secondary，激活状态用 accent-100）

## 八、组件状态设计

### 8.1 卡片组件
- **默认状态**：深色背景，细边框
- **悬浮状态**：渐变边框发光，轻微放大（scale 1.02），阴影扩散
- **点击状态**：短暂的高亮反馈

### 8.2 按钮组件
- **主要按钮**：accent-100 背景色，hover 时发光
- **次要按钮**：透明背景，边框，hover 时填充
- **危险按钮**：红色系，用于删除等危险操作

### 8.3 输入框
- **默认状态**：深色背景，细边框
- **聚焦状态**：发光边框（accent-100 glow）

### 8.4 弹窗
- 毛玻璃背景遮罩
- 居中显示
- 进入动画：scale(0.95) → scale(1) + fadeIn
- ESC 键关闭

| 按键 | 功能 |
|------|------|
| / | 聚焦搜索框 |
| Esc | 关闭当前弹窗/取消搜索 |

## 十、性能要求

- 首屏加载 < 2 秒
- 骨架屏加载动画（与实际布局结构一致）
- 深色主题，动画流畅（60fps）
- 桌面端优先，最小宽度 1024px

---

## 十一、输出要求

设计师需要提供：
1. 完整的高保真设计稿（Figma/Sketch 等）
2. 组件变体说明（不同状态）
3. 色彩 Token 定义
4. 图标使用清单
5. 动画规范（如有特殊动效需求）
