## 简介

### 什么是 Feature Menu？

Feature Menu（功能中心）是一个用于浏览和管理 Claude Code 已安装技能、代理和命令的可视化工具。

当你安装了各种 Claude Code 技能后，时间久了可能记不清有哪些技能、它们各自有什么用。Feature Menu 就是为了解决这个问题而生的——它会自动扫描你的 Claude Code 目录，把所有技能、代理和命令以卡片形式展示出来，并支持搜索、筛选和收藏。

### 核心功能

| 功能 | 说明 |
|------|------|
| 技能浏览 | 卡片式展示所有已安装的技能、代理和命令 |
| 分类筛选 | 按开发框架、数据库、测试、运维等分类快速筛选 |
| 搜索功能 | 关键词搜索，瞬间定位目标技能 |
| 任务推荐 | 告诉它你想做什么，它推荐合适的技能 |
| 收藏功能 | 收藏常用技能，方便快速访问 |
| 最近使用 | 记录最近使用过的技能，一键快速访问 |
| 真实使用量统计 | 通过 Hook 捕获 Claude Code 中真实的技能调用 |
| 详细统计图表 | 柱状图、饼图、排行榜等可视化展示 |

## 安装与启动

### 一键启动（推荐）

只需在 Claude Code 中告诉我"打开功能中心"，我会自动完成以下操作：

1. 启动 Feature Menu 服务
2. 自动打开浏览器访问 http://localhost:3000

无需手动安装任何东西！

### 手动安装（可选）

如果需要手动安装或更新：

#### 第一步：下载插件

通过 SuperPowers Marketplace 或手动克隆：

```bash
git clone https://github.com/ryushowtime/feature-menu.git ~/feature-menu
```

#### 第二步：安装依赖

```bash
cd ~/feature-menu
npm install
```

#### 第三步：一键启动

```bash
cd ~/feature-menu && npm run dev
```

服务启动后，访问 http://localhost:3000

### 指定端口

如果 3000 端口被占用：

```bash
PORT=3001 npm run dev
```

然后访问 http://localhost:3001

## 功能详解

### 1. 技能浏览

在「技能」页面，你可以看到所有已安装的技能。

**浏览方式：**
- 点击左侧「技能」进入技能列表
- 默认按名称字母顺序排列
- 每个技能显示为一张卡片

**技能信息解读：**
| 字段 | 说明 | 示例 |
|------|------|------|
| 名称 | 技能的唯一标识 | `python-developer` |
| 描述 | 技能的用途说明 | "帮你快速构建 Python 应用" |
| 分类 | 所属领域 | 框架、数据库、测试等 |
| 难度 | 适合的使用者级别 | 新手 / 进阶 / 高级 |
| 命令 | 触发技能的命令 | `/python-developer` |

### 2. 分类筛选

侧边栏点击不同分类，或技能卡片上的分类标签，可快速筛选。

**可用分类：**

| 分类 | 图标 | 说明 |
|------|------|------|
| framework | 🧩 | 前端、后端、API 框架 |
| database | 🗄️ | SQL、NoSQL、ORM |
| testing | 🧪 | 单元测试、E2E、测试框架 |
| workflow | ⚙️ | 代理、工作流自动化 |
| business | 💼 | 商业文案、投资人pitch |
| devops | 🚀 | Docker、K8s、CI/CD |
| security | 🔒 | 安全扫描、代码审计 |
| language | 💻 | Go、Python、Java 等 |
| other | 📦 | 不属于以上分类的 |

### 3. 搜索功能

**使用方法：**
- 按 `/` 键快速聚焦搜索框
- 输入关键词实时搜索
- 支持搜索名称和描述

**搜索范围：**
- 技能名称
- 技能描述
- 触发命令

### 4. 收藏功能

**收藏技能：**
1. 鼠标悬停技能卡片
2. 点击右上角的 ⭐ 图标
3. 技能被添加到收藏列表

**访问收藏：**
- 点击侧边栏「❤️ 收藏」
- 查看所有收藏的技能
- 点击任意卡片快速查看详情

**取消收藏：**
- 在收藏列表中点击已收藏的 ⭐ 图标
- 或在技能详情页点击取消收藏

### 5. 最近使用

**记录机制：**
- 通过 Hook 捕获 Claude Code 中真实的技能使用
- 最近使用的技能显示在「最近使用」列表

**用途：**
- 快速重新启用刚用过的技能
- 无需搜索即可找到常用技能

### 6. 任务推荐

**使用场景：**
当你不知道该用哪个技能时，告诉它你想做什么，它会推荐合适的技能。

**使用方法：**
1. 点击侧边栏「🎯 任务推荐」
2. 在文本框中描述你的需求
3. 系统分析并推荐相关技能

**示例输入：**
- "我想写一个 Python Web API"
- "帮我做一个电商网站的前端"
- "需要做数据库迁移"

### 7. 技能详情

**打开详情：**
- 点击技能卡片任意区域
- 弹出详情弹窗

**详情包含：**
| 字段 | 说明 |
|------|------|
| 名称 | 技能完整名称 |
| 描述 | 详细功能说明 |
| 分类 | 所属领域 |
| 难度 | 适合人群 |
| 使用场景 | When to Activate 段落 |
| 触发命令 | 如何在 Claude Code 中调用 |
| 相关技能 | 其他相关技能推荐 |

**使用技能：**
- 点击「开始使用」按钮
- 命令自动复制到剪贴板
- 切换到 Claude Code 粘贴使用

## 使用量统计

### 功能原理

Feature Menu 可以统计你在 Claude Code 中真实使用技能的情况。这通过 Claude Code 的 Hook 机制实现：

1. 当你在 Claude Code 中使用技能（手动或自动触发）
2. Claude Code 内部调用 Skill 工具
3. Hook 捕获调用并记录到 `~/.claude/skills-usage.json`
4. Feature Menu 读取并展示统计数据

### 查看统计数据

启动 Feature Menu 后，可以在两个地方查看统计数据：

**首页仪表盘：**
- 总使用量卡片：显示所有技能的总调用次数
- 最常用列表：展示使用最多的前 5 个技能
- 最近使用列表：显示最近有使用记录的技能

**统计页面（点击侧边栏「📈 统计」）：**
- **总调用次数**：累计所有技能调用
- **已使用技能数**：有多少技能被使用过
- **最常用分类**：使用量最高的技能分类
- **Top 3 柱状图**：使用最多的 3 个技能可视化
- **分类分布饼图**：各分类使用量占比
- **Top 3 排行榜**：详细的使用量排名列表

### 启用/禁用追踪

在统计页面顶部可以一键启用或禁用使用量追踪功能。禁用后 Hook 仍然会记录数据，只是 Feature Menu 不会读取显示。

### 安装 Hook（可选）

使用量统计功能默认已启用。要手动安装或重新安装 Hook：

```bash
# 1. 安装 jq（如果没有）
brew install jq  # macOS
# Ubuntu/Debian: sudo apt install jq
# Windows: 使用 winget 或手动下载

# 2. 运行安装脚本
cd ~/feature-menu
npx ts-node install/install-hook.ts
```

安装脚本会自动：
- 创建 `~/.claude/hooks/` 目录
- 复制 Hook 脚本
- 配置 `~/.claude/settings.json`

## 统计页面详解

点击侧边栏「📈 统计」进入详细统计页面。

### Hook 状态

页面顶部显示 PostToolUse Hook 的启用状态：
- **已启用**：显示绿色徽章，正在从 `~/.claude/skills-usage.json` 读取数据
- **已禁用**：显示红色徽章，暂停追踪

点击按钮可一键切换启用/禁用状态。

### 概览卡片

三个统计卡片显示：
- **总调用次数**：所有技能累计被调用的总次数
- **已使用技能数**：有多少个不同的技能被使用过
- **最常用分类**：使用量占比最高的技能分类

### 可视化图表

**Top 3 柱状图**
- 横向柱状图展示使用次数最多的 3 个技能
- 悬停显示具体使用次数
- 点击可快速跳转到对应技能

**分类分布饼图**
- 环形图展示各分类的使用量占比
- 不同颜色代表不同分类
- 图例显示各分类名称和具体使用次数

### Top 3 排行榜

按使用量排序的详细列表：
- 显示排名（🥇🥈🥉 前三名特殊样式）
- 技能名称和描述
- 分类标签
- 具体使用次数

## 常见问题

### Q: Feature Menu 和 Claude Code 是什么关系？

A: Feature Menu 是一个独立的应用，用于可视化浏览 Claude Code 的技能、代理和命令。它通过扫描本地文件系统来读取这些组件的元数据，不依赖 Claude Code 的内部 API。

### Q: 技能存放在哪里？

A: 技能文件存储在 Claude Code 安装目录下的 `skills/` 文件夹中。Feature Menu 会自动检测以下路径：
- `CLAUDE_CODE_ROOT` 环境变量指定的路径
- `~/.claude`
- `~/everything-claude-code`
- `~/Claude Code`

### Q: 为什么有些技能显示「未分类」？

A: 如果技能名称不匹配任何分类关键词，会被归类为「other」。这是正常现象，不影响技能使用。

### Q: 如何添加新的技能？

A: 将技能文件夹放入 Claude Code 的 `skills/` 目录，Feature Menu 会自动检测并显示，无需重启服务。

### Q: 收藏和最近使用记录保存在哪里？

A: 保存在浏览器本地存储（localStorage）中，清除浏览器数据会丢失。使用量统计数据保存在 `~/.claude/skills-usage.json`。

### Q: 使用量统计是如何工作的？

A: 通过 Claude Code 的 PostToolUse Hook 实现。当技能被使用时（手动或自动），Hook 会将使用记录写入 `~/.claude/skills-usage.json`，Feature Menu 读取并展示。

### Q: 支持多语言界面吗？

A: 目前界面为中文。英文版请参考 [README_EN.md](README_EN.md)。

## 故障排除

### 问题 1：服务启动失败

**症状：** 执行 `npm run dev` 报错

**解决方案：**
```bash
# 1. 检查 Node.js 版本
node --version  # 需要 18.0 或更高

# 2. 删除 node_modules 重新安装
rm -rf node_modules package-lock.json
npm install

# 3. 重新启动
npm run dev
```

### 问题 2：显示「未检测到 Claude Code 目录」

**症状：** 首页显示警告，技能列表为空

**解决方案：**
```bash
# 方法 1：设置环境变量
export CLAUDE_CODE_ROOT=/path/to/your/claude-code-dir

# 方法 2：确保 skills 目录存在
ls ~/.claude/skills/

# 方法 3：启动时指定路径
CLAUDE_CODE_ROOT=~/my-claude npm run dev
```

### 问题 3：端口被占用

**症状：** `Error: Port 3000 is already in use`

**解决方案：**
```bash
# 使用其他端口
PORT=3001 npm run dev
# 然后访问 http://localhost:3001
```

### 问题 4：技能信息不更新

**症状：** 新增的技能没有显示

**解决方案：**
1. 刷新浏览器页面
2. 如果仍然不显示，重启服务
3. 检查技能文件夹是否放在正确位置

### 问题 5：技能卡片显示不完整

**症状：** 某些字段显示为「暂无」

**原因：** 技能的 SKILL.md 文件缺少必要的 frontmatter 字段

**解决方案：**
- 确保技能文件包含 `name` 字段
- 可选字段：`description`、`origin`、`command`

### 问题 6：使用量统计数据为空

**症状：** 技能使用后统计数据没有更新

**解决方案：**
1. 确认 Hook 已安装：检查 `~/.claude/hooks/log-skill-usage.sh` 是否存在
2. 确认 jq 已安装：`jq --version`
3. 检查配置文件：`~/.claude/settings.json` 是否有正确的 Hook 配置
4. 重新运行安装脚本：`npx ts-node install/install-hook.ts`

## 技术细节

### 工作原理

```
用户操作 → Next.js 前端 → API 路由 → 文件系统扫描 → 返回 JSON → 前端渲染
```

1. **前端**：React + Tailwind CSS 构建的 Web 界面
2. **API 路由**：Next.js API Routes 处理扫描请求
3. **扫描器**：读取 skills/agents/commands 目录下的 .md 文件
4. **解析**：提取 YAML frontmatter 元数据
5. **Hook**：PostToolUse Hook 捕获真实技能调用
